# 客户端版本

## 概述

管理游戏客户端版本及更新要求。

**访问路径：** 系统管理 → 客户端版本

## 功能

- 定义最低允许版本
- 设置强制更新提醒
- 分平台版本管理 (iOS, Android, PC)
- 配置更新 URL

## 版本配置字段

| 字段 | 说明 |
|-------|-------------|
| **平台** | iOS, Android, Windows, Mac |
| **最低版本** | 允许运行的最旧版本 |
| **最新版本** | 当前最新版本 |
| **强制更新** | 低于最低版本时是否强制要求更新 |
| **更新 URL** | 商店链接或下载地址 |

## 添加版本的方法

1. 前往 **系统管理** > **客户端版本**。
2. 点击 **添加版本** 按钮。
3. 配置如下：

| 字段 | 类型 | 是否必填 | 说明 |
|-------|------|----------|-------------|
| 平台 | 下拉选择 | 必填 | 目标平台 |
| 版本 | 文本 | 必填 | 版本号字符串 (如 1.2.3) |
| 最低版本 | 文本 | 必填 | 最低版本要求 |
| 强制更新 | 开关 | - | 是否弹出强制更新提醒 |
| 更新 URL | 文本 | - | 下载或商店页面链接 |

4. 点击 **保存**。

## API 检查

客户端在启动时检查版本：

```bash
GET /api/v1/client-version?platform=android&version=1.2.0
```

响应示例：
```json
{
  "needsUpdate": true,
  "forceUpdate": false,
  "latestVersion": "1.3.0",
  "updateUrl": "https://play.google.com/store/apps/..."
}
```
